How to Use this Application

  • You use this application to edit and change the default styles for Primary and Secondary Button Classes that are defined in the CSS File sx_Buttons.css, which you can see in the right text area.
    • However, outside the Public Sphere Applications, you can use it to create completely new Button Classes and styles, based on your Color Schema.
  • With Public Sphere Applications, you might need make two kinds of changes, depending on the darkness or lightness of your Color Schema:
    1. Regarding Change of Colors:
      • Colors are defined as variables of a Color Schema, saved in the CSS-File root_Colors.css.
      • You have to use color variable, not HEX, RGB or other color definitions.
      • However, you can lighten or darken existing colors by adding opacities of white and black on them:
        • To lighten a color, add opacities of white from 10% to 100% by adding the suffix -tint-XX to heir variable name (--color-name-tint-20)
        • To darken a color, add opacities of black from 10% to 100% by adding the suffix -shade-XX to heir variable name (--color-name-shade-20)
        • For the --basic-color, use --basic-tint-XX and --basic-shade-XX. For only this color, you can also add opacities of grey (--basic-tone-XX).
      • Change you Color Schema for alternative colors.
    2. Change the Text Color of Primary Buttons:
      • Colors are defined as variables of a Color Schema in the CSS-File root_Colors.css. Change the Color Schema to change Button Colors.
      • If you Color Shema is dark, you may not nedd to change the Text Color, as it is by default light, usually 90% opacity of white (--tint-90) and 100% (--tint-100) for hover.
      • If your Color Schema is light, use opacities of black, e.g. --shade-80.
    3. Change the Background Color of Primary Buttons:
      • To darken a background color, add black shades from 10% to 100% to its variable name: name-shade-20
      • To lighten a color, add white tints from 10% to 100/ to its variable name: name-tint-20
      • For the basic-color, use basic-tint-XX and basic-shade-XX. You can also desaturate the basic-color by adding grey tones from 10% to 100/ like this: basic-tone-10
  • Obs! You may need to change only the Primary Buttons. Secondary Buttons either inherit your changes of Primary Buttons or have text colors compatible to their background colors.

How to Edit Button Styles

  • For Primary Buttons, just click on a Button, and its styles will be copied in the text area on the right.
  • For Secondary Buttons, just click on the red codes with the class names, above the examples of buttons.
  • Make your changes in the text area and then append them to the Default Style Scheet that is visible in the second text area.
  • Copy then the default style sheet or save it and use it to change the file sx_Buttons.css in your CSS folder.

Primary and Secondary Button Classes

  • Primary Buttons get their colors of the main color palettes of your Color Schema: The Basic color and the Complementary color, and the pairs of Triad, Tetrad and Split colors.
  • Primary Buttons stand alone and can be used as such. But they can also be combined with one or more Secondary Buttons.
  • Secondary Buttons MUST always be used with primary ones to have any effect. The just modify Primary Buttons.

Edit the Styles of Button Classs:

Click a Primary Button or the Class Name of a Secondary Button to copy its styles

Default Button Classes - Copy from the CSS File sx_Buttons.css

Appent your classes to this CSS File, copy and save it to your CSS folder.

1. Primary Button Classes - Buttons that can stand alone or combined with Secondary Buttons Classes. Click on a Button to copy its class to the textarea and edit it

Monochromatic Button Classes: The main construction of Buttons. Hover lightens the background color. Change Text color if needed.

Mixed Color Button Classes: The first two buttons mix the basic with the complementary color. The others mix the left and right side of every color scheme (-1 and -2)

2. Secondary Button Classes - Must always be used with Primary Classes. Click on a red code to copy a class to the textarea and edit it.

Border Buttons: - just add the class .button-border to Primary Classes.

Gradient Buttons: - just add the class .button-gradient to Primary Classes.

Big Buttons: - just add the class .button-big to Primary Classes. Here combined with the secondary classes .button-border .button-gradient

Small Buttons: - just add the class .button-small to Primary Classes. Here combined with the secondary classes .button-arrow, .button-shadow and .button-gradient

From grey Buttons to Colored hover: - just add the class .button-grey-to-color to Primary Classes. Here combined with the secondary class .button-big and some other classes.

Arrow Buttons: - just add the class .button-arrow to Primary Classes.

Shadow Buttons: - just add the class .button-shadow to Primary Classes.

Shadow Buttons with White Background: - just add the class .button-shadow-white to Primary Classes.

Shadow Buttons with White Background and Borders: - just add the class .button-shadow-border to Primary Classes.

3. Secondary Button Classes with Images. Must always be used with Primary Classes.

The link to icon is defined in the content of the pseudoclass :before
For the best effect, use SVG images with transparent background. Click on the red code to copy them to the text area and edit them.

Icon Button: - Just add the class .button-icon to Primary Classes.

Icon Button with Text: - Just add the class .button-icon-left to Primary Classes.

SVG Button: - Just add the class .button-svg to Primary Classes. Here combined with the secondary class .button-shadow

SVG Button with Text: - Just add the class .button-svg-left to Primary Classes.

4. Secondary Button Classes with Images placed within HTML. Must always be used with Primary Classes.

The link to image is defined in the source of the HTML element img, which has to be placed within the class container (INPUT, BUTTON, A, or any HTML element).
For the best effect, use SVG images with transparent background. Click on the red code to copy them to the text area and edit them.

Any Image Button: - Just add the class .button-image to Primary Classes. Here combined with the secondary class .button-shadow

Any Image Button with text: - Just add the class .button-image-left to Primary Classes. Here combined with the secondary class .button-gradient.

Button as Card with Image and Text: - Just add the class .button-image-card to Primary Classes.
Here. the image is placed within a DIV Tag, not a button, together with another DIV for text conten. You can change the height of image (default: 12rem).
Here combined with the secondary class .button-shadow.

.button - This Button Card will expand to the width of its parent container. Its initial height (without text) is determined by the height of the image {height:12rem; width:auto}. To avoid long texts expanding the card vertically, its height is sett to the image height plus 8px for padding and borders {height: calc(12rem + 8px); overflow: hidden;}. So, you will not be able to see the rest of this texts...

.button-complement - This Button Card contains no BUTTON Tag but a DIV with 2 children items, the IMAGE and a DIV containing this text. Its display is FLEX, with the IMAGE item set to FLEX: 0 and the DIV item to FLEX: 1, so it expands to the rest of the container. Its background color is sett to white with opacity 0.9 and its fonts color to black with opacity 0.6.

General Help

Create first a Blueprint Conference Program by filling the information in the above form or by reloading a program saved in the Blueprint Table. Read the Help Information [?] in every Input of the Form for details.

1. Sessions: Start always by scheduling the number of sessions for each conference day, by deleting and adding sessions in the blueprint. When you are ready, save your changes and only then continue with the number of Papers by session.

  • Sessions are distinguished from each other with the succession of parallel colors, gray and white.
  • Every session contains the same number of papers that you have define in the field Papers by Session.
  • When you have defined the number of sessions for every conference day, click on the Button Session ID, on the header of the Table, to automatically set a unique ID Number for every session. ID numbers start automatically from the last Session ID in the database (see on the Top Right of the page).
    • Save your changes in the Blueprint Table when you are ready. You can then recall these changes by Reloading the Blueprint Table.
    • If you change the number of papers in a particular session, use never the Button Session ID again, as it reorganizes sessions with the same number of papers that you defined in the field Papers by Session.

2. Parallel Sessions: You create parallel sessions be copying all papers of a session and moving them all in a separate group, below or above the original session. All copied papers have a common background color so you can distinguish them (se below about papers).

  • Obs! You must still copy the same number of papers as that defined in the field Papers by Session, even if parallel sessions might have different number of papers. You can delete or add papers later (see below) when you are ready with the sessions.
  • When you are ready with parallel session, Click the Button Session ID to get unique ID numbers for all sessions (that is why you must have the same number of papers for every session), save your changes in the Blueprint Table and reload it to get rid of background colors.

3. Papers: When you are ready with the sessions by day and have saved your Program in the Blueprint table, continue with the number of Paper by Session.

  • Obs! Do not use the Button Session ID anymore, as it presupposes that all sessions have the same number of papers!
  • Every row in the Bluprint represents a paper. You add and delete Papers within every session using the icons on the left column.
  • You add a paper by coping an existing row. The copy is inserted below the original row. Both are marked with different background colors. You can then move the copy up and down, but do it only within the same session.
  • When you are ready, save your changes in the Blueprint Table and reload it to get rid of background colors.

4. Change Titles and Time Intervals: You can click on all Table Cells that contain Titles and Time Intervals and change their content.

  • A dialog box appears with a relevant input where you can insert new values. Changes are marked with background colors.
  • You can occasionally save your changes in the Blueprint Table. Reload the table when you are ready, or when you want to get rid of all background colors

5. Export the Blueprint to the Database: When you are ready with all moments, save you changes in the Blueprint Table and click on the button Go to Export Blueprint. This will lead you to a New Page, where you can check and upload the Blueprint Program to the Database

  • The ID numbers of the conference, of its sessions and of its papers start automatically from the last IDs in the Database (see on the Top Right of the page).
    • If changes have been made in the database after you have opened the Export Page, and some of the ID Numbers already exist in the Database, the Blueprint program will not be uploaded.
  • The New Conference and its sessions and papers will all be marked as Hidden, so nothing will be visible on the website. You have than to make your changes from the administration of the website.
    • You can first make the New Conference visible on the website but hold sessions and papers hidden
    • You can then make all sessions visible, when you are ready with them.
    • Finally, you can make the papers visible.

Sessions per Day

Set the number of sessions by day as equal to the Day with the highest number of sessions. If other days contains less Sessions, you can delete them by using the icon [X] on the left column.

  • You delete a session by deleting all Papers belonging to that session.

Alternatively, you can set the lowest or the average number of sessions, but you have then both to delete and to add sessions, which is more complicated.

Papers per Session

Set the number of papers by session as equal to the session with highest number of papers. You can then delete papers from particular Session.

Alternatively, you can set the lowest or the average number of papers, but you have then both to delete and to add papers, which is more complicated.

Minutes Per Paper

Write the everage number of minutes per paper for all sessions. You can then change the minutes per paper for particulat sessions. Please include pauses between Papers

  • Define the everage number of minutes per paper in relation to the number of papers per session and the Start Time of Sessions (see corresponding help about The Start Time of Sessions on the Right).
  • The End Time of Sessions will be automatically counted as Start Time + (Number of Papers * Number of Minutes per Paper).
  • The Start and End Time of papers for every session will be automatically crated by using the number of minutes per paper.
  • However, you can later change the Start and End Time of both Sessions and Papers in the Blueprint, by clicking on the corresponding Table Cell (see General Help).

Start Time for Sessions

Insert Time only for the defined number of Sessions per Day.

  • End Time of every session will be automatically counted as Start Time + (Number of Papers * Minutes per Paper).
  • So, you must take into account the counted End Time of a session when you set the Start Time for the next session.
  • You can then change the Start and End Time of sessions in the Blueprint.

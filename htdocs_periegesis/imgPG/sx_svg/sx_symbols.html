Separate SVG-files included in the Source of Image elements

SVG-attributes cannot be modified by CSS-classes when included as images. You can modify the background of the image.

Set the fill-color of SVG files to black or white with a medium transparancy to change it with the background of the image:

fill="rgba(0,0,0,0.65) or fill="rgba(255,255,255,0.65)

<div class="sx_svg_image">
    <img src="black/sx_home.svg" alt="home" />
</div>

Separate SVG-files included in CSS as backgrounds to pseudo-classes A::before

SVG-attributes cannot be modified by CSS-classes when includes as background images, but you can define and change the color and the content (SVG-image) of the pseudo-class for Default, Hover and Selection.

For SVG files that are used with A-links as A::before pseudo-classes, you do the following:

  • Create a Surounding Path arround the utter bandories of the SVG file in the oposit direction of the rest of paths: M0 0 v32 h32 v-32 z or M0 0 h32 v32 h-32 z, depending on the direction of the other paths.
  • Set the fill-color of the Surounding Path to white (or to the site color): fill="#fff" or style="fill: #fff".
  • Set the background-color of the SVG file to transparent. All paths within the white, Surounding Path of the SVG file will get the transparent background-color.
  • You can now define the default color of the pseudo-classes A::before to whatever you like, and change it for Hover and Selection. The inner paths of the SVG file will get these colors.
  • However, if the background color of the site is other than white, you have to change manually the fill-color of the Surounding Path of every SVG file that you use in links.

Example: 

<svg xmlns="http://www.w3.org/2000/svg" style="background: transaprent; stroke: none; fill: #ffffff; display: block;" viewBox="0 0 32 32">
    <title>sxbg_up_down</title>
    <path d="M0 0 h32v32h-32z M6 14 h20 l-10-10 Z m0 4 l 10 10 10 -10z" />
</svg>

Primary and Secondary Button Classes

  • Colors are defined as variables of a Color Scheme in the CSS-File root_Colors.css. Change your Color Scheme to change Button Colors.
  • Buttons can be constructed with the tags a and button. However, you can use any Tag, if you remove default styles that are not compatible with Button Styles.
  • All Primary Buttons Classes (Monochromatic and Mixed) have two common style definitions (for normal and hover) followed by particular definitions for each one of them.
  • All other buttons are created by adding one or more Secondary Classes to these Primary Classes. However, not all combinations are meaningful.

Changes you can make in the stylesheet sx_Buttons.css

  • Hover lightens the background color of monochromatic Buttons, so the text color must be adapted to it.
  • If your Color Scheme is Dark, use light text colors (shades of white or tints) and lighten them from normal to hover, for example --tint-70 for normal and --tint-90 for hover.
  • If the schema is Light use shades of dark, and darken them from normal to hover, for example, --shade-70 for normal and --shade-90 for hover.
  • For Color schemes in the Middle, you can mix shades of white with shades of black, for example, --tint-80 for normal and --shade-80 for hover.
  • You make these changes in the two common style definitions for all Buttons.
  • In Border cases, set the colors for normal and hover in the particular style definitions of the Button you wish to use, where you can use the shades of respective color. Here, this is down for the gray color.

Primary Button Classes - You use these classes to define the styles of Your Classes.

Monochromatic Button Classes: The main construction of Buttons. Hover lightens the background color. Text color must be adapted to it.

Monochromatic Button Classes with Border:. All colors are set as shades of respective color, independently of common style definitions

Mixed Color Button Classes: The first two buttons mix the basic with the complementary color (-bc and -cb) The others mix the left and right side of every color scheme (-lr and -rl)

Secondary Button Classes - You cannot use these classes for style definition of your classes, just add their name to your class names.

Gradient Buttons: - just add the class .button-gradient to Primary Classes.

Gradient Buttons with Borders: - just add the class .button-gradient-border to Primary Classes. Here combined with the secondary class .button-big
If you combine it with Pramary Classes with borders, .button-border-, the gradient effects will be lighter (see bellow).

Gradient Buttons with Borders: - The above class combine with Pramary Classes with borders, .button-border-, gives lighter gradient effects.

Gradient Buttons with Borders. from gray to color hover: - just add the class .button-gradient-to-color to Primary Classes. Here combined with the secondary class .button-big
If you combine it with Pramary Classes with borders, .button-border-, the gradient effects will be lighter (see bellow).

Arrow Buttons: - just add the class .button-arrow to Primary Classes.

Shadow Buttons: - just add the class .button-shadow to Primary Classes.

Shadow Buttons with White Background: - just add the class .button-shadow-white to Primary Classes.

Shadow Buttons with White White Background with borders: - just add the class .button-shadow-border to Primary Classes.

Big Buttons: - just add the class .button-big to Primary Classes. Here combined with the secondary class .button-gradient

Small Buttons: - just add the class .button-small to Primary Classes. Here combined with the secondary classes .button-arrow, .button-shadow and .button-gradient

Button as Card with Image and Text: The link to image is defined in the source of the img Tag. Here. the image is placed within a DIV Tag, not a button, together with another DIV for text conten.
Just add the class .button-image-card to Primary Classes (use SVG or PNG images with transparent background). Here combined with the secondary class .button-shadow. You can change the height of image (default: 12rem)

.button - This Button Card will expand to the width of its parent container. Its initial height (without text) is determined by the height of the image {height:12rem; width:auto}. To avoid long texts expanding the card vertically, its height is sett to the image height plus 8px for padding and borders {height: calc(12rem + 8px); overflow: hidden;}. So, you will not be able to see the rest of this texts...

.button-complement - This Button Card contains no BUTTON Tag but a DIV with 2 children items, the IMAGE and a DIV containing this text. Its display is FLEX, with the IMAGE item set to FLEX: 0 and the DIV item to FLEX: 1, so it expands to the rest of the container. Its background color is sett to white with opacity 0.9 and its fonts color to black with opacity 0.6.

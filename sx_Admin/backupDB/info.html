Backup to Restore or Copy a Database Schema

Select Tables

  • You can backup a Sole Table, any Set of Tables or the Entire Database.
  • If you select a Sole Table or a Set of Tables, please also include all other tables that might be related to the selected ones.
  • Usually, although not necessary and not exclusively, you might include all tables within a successive color group.

The Name of the Backup File

  • The default name of the backup file is the Database Name followed by:
    • The content type of the Backup File (see information besides):
      • Contents: Truncate tables and import backed up contents.
      • Structures: Statements for the creation of Database and its Tables' Structures, without contents
      • Both: Both Structures and Contents.
    • A suffix with the current Data and Time.
    • For example: database_name_Both_2021-11-14_17-05-28.sql
  • If you select anly a set of tables, the first table name and the number of selected tables will be automatically added to the Database Name, together with the above suffix (database_name_Contents_text_authors_7_tables_2021-11-14_17-21-15.sql).
  • However, You can set whatever name you like - and also delete the suffix.

The Content of the Backup File

  • Backup Only Contents
    • The default Backup File includes the SQL Statements:
      • TRANCATE TABLE, to remove all contents of tables but keep their structure, and
      • INSERT INTO ... VALUES, to restore contents with the backed up ones.
  • Backup Both Table Structures and Contents
    • Click on this box to include in the Backup File the SQL statements:
      • CREATE DATABASE IF NOT EXISTS, to create the Database Schema, if it does not exist.
        • However, you can also change the name of the current Schema (New Database Name) and create copies of your current database (a Backup Databases rather the only a file).
        • You can then, instead of restoring the current database, redirect the connection of your site to the Backup Database.
      • DROP TABLE IF EXISTS, which deletes not only the content but also the structure of table.
      • CREATE TABLE, which recreates the structure of the deleted table.
      • INSERT INTO ... VALUES, which adds the backed up content to the above table.
  • Backup Only Table Structures
    • Click on this box if you want to create another database, for another project, with the same Table Structures and the same or different Database (Shema) Name as the current one, however, without content.
    • Define a New Database Name if you want to change its name.
    • The SQL statements include all the above-mentioned statements, except the last one: INSERT INTO ... VALUES

SQL, GZIP and ZIP

  • By default, the Backup file is saved in SQL Format. For large databases, please select the GZIP Compression.
  • You can Restore your Database (or create a copy) by using Backup File of both SQL and GZIP format - The program automatically recognises the format of the file and imports it.
  • If you select to Download the SQL Backup File on you local computer, it will be compressed in ZIP Format by default.
